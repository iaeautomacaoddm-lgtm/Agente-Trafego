2. OPÇÕES DE INTEGRAÇÃO DASHBOARD ↔ HERMES
================================================================================

--------------------------------------------------------------------------------
OPÇÃO 1: Hermes CLI (Recomendado para início)
--------------------------------------------------------------------------------

Prós:
✅ Mais simples de implementar
✅ Não precisa de servidor extra
✅ Funciona offline

Contras:
❌ Sincrono (bloqueia até terminar)
❌ Sem atualizações em tempo real

Implementação:
```javascript
// No backend Node.js do dashboard (ou Electron)
const { spawn } = require('child_process');

function callHermes(skillName, input) {
  return new Promise((resolve, reject) => {
    const hermes = spawn('hermes', [
      '--skill', skillName,
      '--input', JSON.stringify(input)
    ]);
    
    let output = '';
    hermes.stdout.on('data', (data) => output += data);
    hermes.on('close', (code) => {
      if (code === 0) resolve(JSON.parse(output));
      else reject(new Error(`Hermes exited with code ${code}`));
    });
  });
}
```

--------------------------------------------------------------------------------
OPÇÃO 2: Hermes API REST (Para produção)
--------------------------------------------------------------------------------

Prós:
✅ Assíncrono
✅ Pode consultar status
✅ Múltiplos clientes simultâneos

Contras:
❌ Precisa rodar servidor Hermes
❌ Mais complexo de configurar

Implementação:
```javascript
// AgentService.js
const HERMES_API = 'http://localhost:3001';

async function executeAgent(agentId, input) {
  const agent = DDM_AGENTS.find(a => a.id === agentId);
  
  const response = await fetch(`${HERMES_API}/execute`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      skill: agent.skill,
      input: input
    })
  });
  
  return response.json();
}
```

--------------------------------------------------------------------------------
OPÇÃO 3: n8n como Middleware (Para automações complexas)
--------------------------------------------------------------------------------

Prós:
✅ Interface visual para workflows
✅ Integração nativa com Meta Ads API
✅ Agendamento de tarefas
✅ Logs e monitoramento

Contras:
❌ Mais uma dependência
❌ Overhead para casos simples

Quando usar n8n:
• Automação de coleta de métricas do Meta Ads
• Agendamento de campanhas
• Notificações (Slack, Email, Telegram)
• Integração com outras ferramentas (CRM, Google Sheets)

Quando NÃO usar n8n:
• Execução simples de agentes
• Prototipagem rápida
• Ambiente local/desenvolvimento
